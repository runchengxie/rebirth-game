# 重生、调查与跨周目系统

本文档说明当前接入主游戏的重生元状态、关键月份调查、年度流程图和研究室中枢。系统把未来记忆、失败复盘、人物关系、研究方法和办公室痕迹连接起来。2023、2024 和 `demo` 继续沿用原流程，新增调查优先服务 2025 主线。

## 核心循环

2025 年的关键月份会在研究选择前提供有限时间块。玩家可以把时间投入公开材料、财务模型、产业访谈、量化信号、风险讨论、档案审计或休息。

调查节点包含以下约束：

- 时间成本。
- 前置调查。
- 周目要求。
- 记忆钥匙要求。
- 研究捷径要求。
- 研究室整理结果。

完成调查会留下待验证线索、已验证证据、反例或状态记录。提交研究方案时，线索会按照研究方法影响证据、清晰度、风险意识、反思价值和疲劳。

玩家可以提前提交方案。省下时间会保留较低疲劳，证据不足也会进入月度评分和后续人物反馈。

## 关键月份

| 月份 | 主题 | 主要冲突 | 跨周目作用 |
|---|---|---|---|
| 一月 | 未来记忆审计 | 技术突破如何传导到收入、利润和竞争 | 建立单位经济性、失败样本和观察仓入口 |
| 四月 | 结果正确与证据完整 | 一季报证明结论后，当时证据是否已经充分 | 使用因果缺口审计事后正确偏差 |
| 七月 | 产业链与资金分化 | 基本面兑现和资金选择出现时间差 | 使用失败样本和捷径完成分层再平衡 |
| 九月 | 规则变化与组织权限 | 新规改变因子、成本和风险边界 | 把同事关系转化为闭门数据权限与共同校准协议 |
| 十二月 | 年度档案与真相路线 | 未来记忆来自亲历、标题或事后解释 | 审计全年档案并提出被污染的未来假设 |

其他月份继续承担资源积累、人物推进、方法训练和研究室物件增长。关键月份读取这些积累，避免每一话都扩展成相同规模的调查关卡。

## 记忆钥匙

完成年度结局并开始下一周目时，系统根据整年行为发放记忆钥匙：

| 钥匙 | 来源 | 下一周目作用 |
|---|---|---|
| 因果缺口 | 空降结论或低推理评分 | 解锁单位经济性、事后正确审计和记忆来源比对 |
| 样本污染 | 长期使用量化方法或追逐热度 | 解锁原始失败区间、失败应用样本和全年反事实 |
| 身体记忆 | 高疲劳或透支路线 | 首次主动休息不消耗时间，并进入可持续边界审计 |
| 机会成本 | 长期风险管理或平衡路线 | 解锁观察仓、分阶段验证和产业链再平衡 |

钥匙不会直接标记正确答案。它改变玩家可以调查的问题和能够进入的剧情分支。

## 研究捷径

人物关系和长期方法会在周目结束时转化为研究捷径：

- 林若宁的产业联系人簿降低产业访谈成本。
- 赵承宇的自动回测管线降低因子调查成本，并开放九月闭门失败样本。
- 周明昭的风险审查模板降低风险讨论成本。

七月的完整混合路线需要自动回测管线。普通时间预算无法同时完成业务热力图、失败样本和分层再平衡。捷径因此会改变可行路线，不只缩短动画或增加结局分数。

## 研究室中枢

档案抽屉新增研究室标签。五类物件会读取本周目和跨周目状态：

| 物件 | 状态来源 | 整理结果 |
|---|---|---|
| 便签墙 | 基本面研究与协作次数 | 回看证据缺口、承诺和人物回调 |
| 白板 | 框架型研究次数 | 连接跨月假设，第二周目可能发现未重置字迹 |
| 咖啡杯 | 深度研究与加班次数 | 检查透支并恢复少量疲劳和生活平衡 |
| 研究档案柜 | 已完成月份 | 按假设和反例重排记录，开放十二月档案审计 |
| 研究捷径板 | 跨周目捷径 | 把联系人、回测管线和风险模板可视化 |

研究室整理只执行一次，并写入稳定旗标。后续月份可以使用这些旗标触发回调或调查前置条件。

## 年度流程图与已读跳过

档案抽屉新增年度流程标签。流程图显示：

- 十二个月的完成、当前和未进入状态。
- 五个关键月份及其谜题主题。
- 已完成研究方案和月度评级。
- 调查路径与线索数量。
- 方法标签和空降结论记录。

对白在玩家实际推进后才写入已读记录。第二周目以后可以连续跳过已经读过的对白。跳过会停在首个新对白或未结算研究选择，也不会跨过月份边界。

## 真相路线

第二周目取得因果缺口和样本污染后，十二月会出现未来记忆来源审计。玩家还需要：

1. 在研究室整理全年档案。
2. 比对未来记忆和后来标题。
3. 恢复失败月份与被筛选样本。
4. 把身体状态或机会成本写进框架。
5. 使用全部线索形成被污染的未来假设。

完成调查后会开放年度策略会专属方案。该方案写入 `rebirth_truth_route`，结局面板优先显示真相结局。

## 存储与迁移

单周目状态继续保存在：

```text
rebirthGameState:v2:<year>
```

跨周目状态保存在：

```text
rebirthMeta:v2:<year>
```

系统会继续读取旧的 `rebirthMeta:v1:<year>`。旧存档中的单个一月调查会迁移到 `investigations["2025-01"]`，钥匙、捷径、异常和历次结局保持不变。

重新开始当前周目会清空本轮调查进度，保留已读节点、记忆钥匙、研究捷径、系统异常、研究室发现和历次复盘。完成十二话并选择带着记忆重生，才会增加周目编号并结算新的跨周目奖励。

## 代码位置

- `src/game/rebirth.ts`：元状态、存档迁移、调查结算和周目奖励。
- `src/game/rebirthInvestigationData.ts`：关键月份调查章节、节点与线索。
- `src/game/rebirthSpecialDecisions.ts`：调查解锁的研究方案。
- `src/game/rebirthDecisionBonus.ts`：线索与研究方法的评分加成规则。
- `src/game/rebirthBranches.ts`：读取周目和记忆钥匙的剧情分支。
- `src/game/rebirthFlow.ts`：年度流程图、已读记录和连续跳过。
- `src/game/rebirthOffice.ts`：研究室物件、发现和状态变化。
- `src/components/RebirthPanel.tsx`：调查、流程图、研究室和跨周目档案界面。
- `src/rebirth.css` 与 `src/rebirth-v2.css`：调查和中枢界面样式。
- `src/app/useGameController.ts`：同步单周目与跨周目状态。
