# 稳定性、浏览器回归与无障碍

本文记录运行时恢复、浏览器端回归测试、键盘操作和社区内容包安全边界。功能设计与模式契约继续见 `platform-modes.md`。

## 运行时恢复

顶层应用由 `AppErrorBoundary` 保护。懒加载组件或渲染过程抛出未处理错误时，页面显示恢复界面，保留浏览器本地存档，并提供重新载入和返回年度剧情两个入口。

错误边界会派发 `rebirth-app-error` 浏览器事件，事件包含错误消息和 React 组件栈。项目当前不会自动上传该信息。

错误边界无法捕获事件处理函数、异步回调和浏览器外部脚本中的全部错误。新增异步流程时仍需要在业务边界处理失败状态。

## 键盘和焦点

页面提供「跳到主要内容」链接，键盘用户可以绕过底部模式导航。

档案抽屉遵循模态对话框行为：

- 打开后焦点移到关闭按钮。
- `Tab` 和 `Shift+Tab` 保持在抽屉内部。
- `Escape` 关闭抽屉。
- 关闭后焦点回到打开档案的按钮。
- 标签页支持左右方向键切换。
- 当前标签页是唯一进入普通 Tab 顺序的标签按钮。

新增模态界面时应复用相同的焦点进入、限制、关闭和恢复规则。

## 单元测试与浏览器测试

Vitest 只运行 `src/` 下的单元和集成测试。`vitest.config.ts` 明确排除 `scripts/e2e/`，避免两个测试运行器收集同一文件。

浏览器测试位于：

```text
scripts/e2e/platform.spec.js
```

Playwright 配置位于：

```text
scripts/playwright.config.js
```

首次在本地运行：

```bash
npm run e2e:prepare
npm run test:e2e
```

`e2e:prepare` 使用固定版本临时安装 Playwright 和 axe，并下载 Chromium。依赖不会写入 `package.json` 或 `package-lock.json`。这避免普通开发和生产构建承担浏览器二进制依赖，同时让 CI 使用可重复的版本。

## 浏览器回归范围

当前 Chromium 回归覆盖：

- 年度剧情的跳过导航。
- 档案抽屉打开、关闭和焦点恢复。
- 独立投委会完成五轮答辩并保存历史。
- 每日挑战保存首次结果，练习不覆盖首次记录。
- 内容工坊保存内容包后，案例进入投委会案例库。
- 平台模式加载失败时显示恢复界面。
- 四种模式的严重和致命 axe 规则。

Playwright 失败时，CI 保存 HTML 报告、截图、视频和 trace，保留七天。

## 自动无障碍检查边界

axe 检查严重和致命问题。当前自动检查关闭 `color-contrast` 规则，因为动态主题、背景图和半透明叠层会产生需要人工判断的结果。

以下内容仍需要人工检查：

- 浅色和深色主题的文字对比度。
- 200% 字体缩放和浏览器缩放。
- Windows 高对比模式。
- `prefers-reduced-motion`。
- VoiceOver、NVDA 或其他真实读屏器。
- 小屏触控目标和软键盘遮挡。
- 中文语音提示的停顿和朗读顺序。

自动检查通过只说明没有命中当前规则集中的严重问题，不代表获得完整无障碍认证。

## 社区内容包安全边界

内容包继续使用版本 1 格式，并增加以下限制：

- 单个 JSON 内容包最多 256 KiB。
- 每个内容包最多 20 个案例。
- 每个案例最多 8 个研究方案，至少 2 个。
- 案例 ID 和方案 ID 在各自作用域内必须唯一。
- 标题、作者、说明、假设、业务事实和方案文本有长度上限。
- `updatedAt` 必须是可解析的日期时间。
- 评分字段必须是有限数字并位于原有范围内。

解析器在 `JSON.parse` 之前检查 UTF-8 字节数，避免超大输入先占用过多内存。保存到本地案例库前会再次检查序列化后的体积。

这些限制保护浏览器存储和投委会案例库，不构成内容事实审核。社区案例的来源、版权和金融事实仍由作者与导入者判断。

## CI 和发布

拉取请求和 `main` 更新先运行静态质量任务，包括 lint、类型检查、Vitest、前端契约、品牌校验、生产构建和包体预算。

静态质量通过后，独立 `e2e` 任务安装固定版本的浏览器测试运行时并执行 Chromium 回归。GitHub Pages 发布依赖静态质量和浏览器回归同时通过。

单元测试失败时保存 `vitest.log` 三天。浏览器回归失败时保存 Playwright 诊断七天。
